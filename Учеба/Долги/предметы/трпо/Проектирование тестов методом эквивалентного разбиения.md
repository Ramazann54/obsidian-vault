Все множество входных данных разбивается на **классы эквивалентности**.  
Предполагается, что если один представитель класса выявляет ошибку, то и другие значения этого класса, вероятно, поведут себя аналогично.

Значит, не нужно тестировать каждое значение — достаточно взять **по одному или нескольку представителей** из каждого класса.

## Как строят тесты

1. Определяют входные данные и ограничения.
2. Разбивают область значений на классы.
3. Выбирают по одному представителю из каждого класса.
4. Формируют тесты.

## Преимущества

- существенно уменьшает количество тестов;
- позволяет системно покрыть разные типы входов;
- особенно полезен для форм, полей, параметров, интерфейсов.

## Недостатки

- зависит от правильности выделения классов;
- может не поймать ошибки на границах классов.

Поэтому его обычно дополняют **анализом граничных значений**.

## Вывод

Эквивалентное разбиение — это способ сократить число тестов без резкой потери качества, выбирая представителей из типовых групп входных данных.
