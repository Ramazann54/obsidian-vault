Критерий покрытия решений требует, чтобы **каждое решение в программе принимало все возможные исходы хотя бы один раз**.

Для обычного `if` это означает:

- ветка `истина` должна быть выполнена;
- ветка `ложь` тоже должна быть выполнена.

Этот критерий часто называют **branch coverage**.

## Что считается решением

Решение — это точка выбора в программе:

- `if`;
- `while`;
- `for` с условием;
- `case/switch`.

## Пример

if x > 0 then  
    y := 1  
else  
    y := -1

Для покрытия решений нужны минимум два теста:

- `x = 1` → условие истинно;
- `x = 0` → условие ложно.

## Чем покрытие решений сильнее покрытия операторов

Если достигнуто покрытие решений, то обычно покрываются и операторы обеих ветвей.  
Но главное — проверяется **сам факт выбора между ветвями**, а не только выполнение строк.

## Недостаток

Покрытие решений не гарантирует, что **каждое простое условие внутри составного решения** проверено отдельно.

Пример:

if (A or B) then  
    S1  
else  
    S2

Тесты:

- `A=true, B=false`
- `A=false, B=false`

Здесь решение покрыто:

- один раз истина,
- один раз ложь.

Но условие `B=true` ни разу не проверялось.

## Вывод

Покрытие решений лучше, чем покрытие операторов, потому что проверяет обе стороны ветвления. Но и этого может быть недостаточно для сложных логических выражений.
